## Gate A — per-test framework tax

Method: marginal cost via (T800 − T200)/600 on trivial `expect(true)` tests
under tests/Feature/WarpBench (transaction-wrapped, realistic tax).

| Mode | Marginal ms/test | 200-test run (s) | 800-test run (s) |
|---|---|---|---|
| Classic (WARP_WARM=0) | 53.57 | 26.31 | 58.45 |
| Warm (WARP_WARM=1) | 0.55 | 13.47 | 13.80 |

Raw benchmark output (verbatim, `bench/warm-tax.sh ~/EA/projects/yardpilot-warp`):

```
WARP_WARM=0  marginal 53.57 ms/test  (200 tests: 26.31s, 800 tests: 58.45s)
WARP_WARM=1  marginal 0.55 ms/test   (200 tests: 13.47s, 800 tests: 13.80s)
```

Machine load during run: quiet — no concurrent do-work Pest runner active on the
yardpilot main checkout (0 competing pest/phpunit processes; system load average
~5.4 on a multi-core M-series Mac, i.e. background only). Numbers are therefore
representative and, if anything, conservative — the warm figure has ample headroom.

Result: warm marginal 0.55 ms/test (spec target ~5ms, comfortably beaten);
classic 53.57 ms/test ⇒ classic is 97.4× warm, far exceeding the 5× requirement.
Contingency (sentinel `HermeticitySentinel::check` config-hash profiling) was
**not required** — warm is an order of magnitude under the 10ms threshold, so no
lazy-hash change to `src/Sentinel/HermeticitySentinel.php` was made.

**Gate A verdict: PASS (threshold: warm ≤ 10ms and classic ≥ 5× warm; spec target ~5ms)**
